## 1. 서비스 한 줄 요약

> **인디게임닷컴 공식 파트너로서, 인디게임 개발자에게 ASO 분석과 바로 적용 가능한 완성물을 전달하는 컨시어지 서비스**

## 2. 핵심 원칙 (절대 잊지 말 것)

| 원칙 | 의미 |
|------|------|
| **바로 적용 가능** | 리포트가 아닌 완성물 (텍스트 + 스크린샷 이미지) |
| **공신력 전면** | 인디게임닷컴의 미디어 지위 + 정무식/임재청 네트워크 |
| **진실된 파트너** | "싸구려 자동화" 아님, 결과물 품질로 말한다 |
| **한국어 기반** | K-인디 우선, 해외 인디는 후속 Phase |
| **관계 중심** | 일회성 거래 아님, 지속적 관계 |
| **시간차 전달** | 내부 처리가 빠르더라도 5영업일 후 전달 |
| **"AI" 언급 금지** | 마케팅/고객 대면에서 내부 기술 노출 안 함 |

### 정무식/임재청의 실제 역할 (중요)

**ASO 서비스에서 두 분은 ASO 전문가가 아니다.**
- ASO 분석/생성은 내부 시스템이 수행
- 두 분의 역할: 공신력 + 게임 업계 네트워크 + 형식적 QC
- 편집장 코멘트는 "환영/격려" 수준 (ASO 조언 아님)
- 주문당 업무 시간: 10~15분

**보도자료 서비스(Phase 2)에서는 다르다:**
- 임재청의 기자 경력 + 반다이남코 경력 + 미디어 네트워크가 실질 가치
- 여기서만 실제 검수/편집/승인 업무 존재

## 3. Phase 1 확정 스펙 전체 요약

### 서비스 범위
- **플랫폼**: Google Play only (MVP)
- **대상**: 국내 인디게임 개발자 (해외는 후속 Phase)
- **결과물**: 텍스트 완성본 + 스토어 규격 스크린샷 + 편집장 코멘트

### 입력 요구사항 (개발자 제공)
**필수**:
- Google Play 스토어 링크 (이미 런칭한 경우)
- 원본 스크린샷 5장 이상 (1920x1080+)
- 고해상도 게임 로고
- 핵심 특징 3가지 서술
- 게임 장르 / 타겟 시장

**선택** (많을수록 좋음):
- 트레일러/게임플레이 영상, 캐릭터 아트, UI 에셋, 개발사 소개, 기타 자료

### 결과물 (5영업일 후 전달)
**텍스트**:
- 게임 제목 대안 3개
- 서브타이틀 대안 3개
- 스토어 소개문구 완성본 (첫 252자 최적화)
- 프로모션 텍스트 (iOS용, 해당 시)
- 키워드 30~50개 (우선순위 포함)

**이미지**:
- 스토어 스크린샷 5~8장 (Google Play 규격)
- 아이콘 피드백/개선안

**분석**:
- 장르별 Top 10 벤치마크 비교
- 경쟁 게임 5개 분석
- ASO 점수 (항목별 100점)
- 개선 우선순위

**편집장 코멘트**: 임재청의 개인 맞춤 코멘트

### 수정 정책
- 14일 내 **2회 무료 수정**
- 3회차부터 유료

### 가격 (잠정)
- 스타트 (Google Play 1개 플랫폼): 20~25만원
- 기본 (iOS+Android): 35~40만원 ← Phase 1.5부터
- 글로벌: 60~80만원 ← 후속 Phase

## 5. 운영 플로우 (정무식 & 임재청이 실제로 일하는 방식)

### 핵심 질문: 백오피스 vs 애플리케이션?

**결론: 웹 기반 백오피스(관리자 대시보드)를 만든다**

이유:
- 두 분이 비개발자이므로 설치/설정 없이 웹 로그인만으로 사용 가능
- 주문/고객/결과물이 한 곳에 모여 있어 업무 효율
- 스마트폰/태블릿/PC 어디서나 접근
- 기존 `indiegame-opal.vercel.app` 프로토타입과 같은 도메인에서 `/admin` 경로로 통합

### 백오피스 구성 (필수 기능)

```
/admin 로그인
  │
  ├── [대시보드] 오늘의 주문 / 진행 중 / 납기 임박 요약
  │
  ├── [주문 관리]
  │   ├── 주문 목록 (상태별 필터: 접수/진행중/검수/완료)
  │   ├── 주문 상세
  │   │   ├── 고객 정보
  │   │   ├── 게임 정보 + 스토어 링크
  │   │   ├── 업로드된 material 다운로드
  │   │   ├── [AI 분석 실행] 버튼
  │   │   ├── AI 생성 초안 (텍스트)
  │   │   ├── 초안 편집 가능 (편집장 스타일 적용)
  │   │   ├── 스크린샷 결과물 업로드 (Figma/Canva에서 제작 후)
  │   │   ├── [편집장 코멘트 작성]
  │   │   ├── 상태 변경 (진행중 → 검수완료 → 전달)
  │   │   └── [고객에게 전달] 버튼
  │   │
  │   └── 수정 요청 관리
  │
  ├── [매체/데이터 관리] (Phase 2부터)
  │
  └── [설정]
      ├── 담당자별 알림 설정
      └── 편집장 스타일 프롬프트 업데이트
```

### 정무식 (발행인)의 역할

```
일상 업무:
- 중요 고객 (규모 있는 개발사) 초기 상담
- Founding Partner 제안 (개인 채널)
- 브랜드 관련 의사결정
- 편집장이 판단 어려운 케이스 자문

백오피스 사용:
- 주 2~3회 대시보드 확인
- 중요 주문만 상세 리뷰
- 월간 성과 리포트 확인
```

### 임재청 (편집장)의 역할 — 주력 운영자

```
일상 업무 (주문 1건당 실제 작업 시간 10~15분):

Day 1 (주문 접수 시):
- 알림 이메일 수신 → 백오피스 접속
- 고객 material 확인 (3~5분)
- 필요 시 고객에게 추가 질문 이메일 발송

Day 2-4 (자동 처리됨):
- 내부 시스템이 전체 결과물 자동 생성
  (텍스트 + 이미지 + 가이드)
- 임재청은 이 단계에 개입 불필요

Day 5 (QC & 전달):
- 결과물 형식적 QC (오탈자, 명백한 오류만) (5~10분)
- (선택) 편집장 환영 메시지 작성 (3~5분)
- [고객에게 전달] 버튼 클릭
- 자동으로 고객 이메일 발송 + 대시보드 업데이트
```

### 실제 사용 도구

| 용도 | 도구 |
|------|------|
| 주문 관리 / 고객 소통 | **백오피스** (우리가 만들 것) |
| 분석 실행 | **백오피스 내 자동 처리** (백그라운드 자동) |
| 결과물 QC | **백오피스 미리보기 + 간단 수정** |
| 고객과 이메일 소통 | **이메일 (Resend 연동)** |

→ 두 분은 **백오피스만** 사용. Figma/Canva 등 외부 도구 불필요.

### 디자이너 없음 (핵심 설계)

스크린샷 제작은 **HTML 템플릿 + Puppeteer 자동 렌더링**으로 완전 자동화.
- 사전 제작된 장르별 템플릿 10~15개
- 원본 스크린샷 + 자동 생성 카피 → 자동 렌더링
- 자동 생성 품질이 아쉬우면 **상세 가이드 문서** 동시 제공

→ 외주 디자이너 상시 투입 없음, 임재청도 직접 제작하지 않음

## 7. 기술 스택 (확정)

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js 16 | 기존 프로토타입 연속성 |

| 결제 | 추후 결정 (초기엔 무통장 입금으로 시작 가능) | - |
| 배포 | Vercel | 기존 연속성 |

## 8. 구현 단계 (실제 개발 순서)

### Stage 0: 준비 (사용자 할 일)
- [ ] Anthropic API 키 발급 (console.anthropic.com)
- [ ] Supabase 프로젝트 생성 (supabase.com)
- [ ] Resend 계정 생성 (resend.com)
- [ ] 각 서비스의 API 키 공유
- [x] 서비스 브랜드명 결정 → **blockbusterlab**
- [ ] blockbusterlab.com 도메인 가용성 확인 (선택)

### Stage 1: 기반 인프라 (1~2일)
- [ ] 환경변수 파일 설정 (.env.local)
- [ ] Supabase 연결 + 인증 설정

### 테스트 진행 방식

```
1. 기본 기능 개발 완료 (Stage 1~5)
    ↓
2. 루노소프트에 테스트 요청 (6개 게임 중 1개 선정)
    - 추천: Love is 또는 브레드틀린그림찾기 (캐주얼/퍼즐 - 우리 벤치마크 DB 풍부)
    ↓
3. 실제 신청 폼을 통해 접수
    - material 수집 (기존 스크린샷, 로고 등)
    ↓
4. 전체 플로우 실제 실행
    - AI 분석 → 편집장 검수 → 스크린샷 제작 → 전달
    ↓
5. 결과물 품질 평가
    - 실제 Google Play에 적용
    - 적용 전후 30일 추적 (다운로드, 검색 노출 변화)
    ↓
6. 피드백 수렴 → 서비스 개선 → 다음 게임 진행
```
