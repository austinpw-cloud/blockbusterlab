## 1. 서비스 개요

**ASO 분석 & 최적화 서비스** — 인디게임 개발자가 게임 정보와 원본 material을 전달하면, 장르별 Top 게임 벤치마크와 비교 분석하여 **바로 스토어에 적용 가능한 완성된 결과물**을 제작해 전달.

### 포지셔닝
- 인디게임닷컴의 K-인디 전문 미디어 지위 + 정무식/임재청의 게임 업계 네트워크 공신력
- 리포트/가이드가 아닌 **바로 적용 가능한 완성물** (텍스트 + 이미지)
- 한국어 기반 인디게임 전문 ASO 서비스 — 국내 영어권 ASO 에이전시(Appfigures·Apptweak·Phiture 등)와 달리 한국어 스토어 카피 품질·K-인디 맥락에 특화. (※ 이 주장의 시장 조사 근거는 별도 검증 필요 항목)
- **마케팅 메시지에 "AI" 언급 없음** — 결과물 품질로 말한다

## 2. 입력 요구사항 (개발자가 전달할 것)

### 필수 항목

| 항목 | 상세 |
|------|------|
| **게임 스토어 링크** | Google Play URL 또는 Apple App Store URL (또는 둘 다). 출시한 스토어 기준 하나 이상 필수. 둘 다 없으면 파일 업로드 필수 |
| **원본 스크린샷** | 최소 **5장**, **1920x1080 이상** 해상도 |
| **게임 로고** | 고해상도 원본 파일 |
| **핵심 특징 3가지** | 게임의 차별화 포인트 |
| **게임 장르** | 자동 분류 가능하지만 개발자 명시 권장 |
| **타겟 시장** | 한국/글로벌/일본/중국 등 |

### 선택 항목 (많을수록 좋음)

- 게임 트레일러/게임플레이 영상
- 캐릭터 아트/일러스트
- 게임 UI 에셋
- 개발사 소개 자료
- 수상 경력/미디어 노출 자료
- 경쟁작 참고 자료
- 특별히 강조하고 싶은 포인트
- 피해야 할 표현/방향성

## 3. 결과물 (Deliverables)

### 텍스트 결과물

| 항목 | 내용 |
|------|------|
| **게임 제목 대안** | 3개 (완성본, 바로 적용 가능) |
| **서브타이틀 대안** | 3개 (완성본) |
| **스토어 소개문구** | 완성본 (첫 252자 최적화) |
| **키워드 리스트** | 30~50개 (우선순위 포함) |
| **프로모션 해시태그** | SNS/커뮤니티용 추천 태그 |

### 이미지 결과물 ★ 핵심 차별점

**두 가지 형태로 동시 제공**:

#### A. Composite 스크린샷 (주력, §5 참조)
- 원본 게임 스크린샷을 1080×1920 전체 배경으로 사용
- 오버레이(텍스트·강조)만 디자인해 합성
- Google Play 스크린샷 규격 맞춤
- 5~8장 PNG 파일
- 각 슬롯에 ASO 키워드 반영된 오버레이 텍스트

#### B. 상세 제작 가이드 (보장)
- 슬롯별 상세 지침 (개발자가 직접 제작 원하는 경우)
- 각 슬롯: 원본 스크린샷 선택 권장 + 카피 완성본 + 폰트/색상/크기 + 레이아웃 와이어프레임 + 참고 예시
- 단계별 Canva 제작 가이드 (30분~1시간 내 직접 제작 가능 수준)

### 분석 결과물

| 항목 | 내용 |
|------|------|
| **장르별 Top 10 벤치마크** | 해당 장르 상위 게임 비교 분석 |
| **경쟁 게임 분석** | 유사 장르 5개 게임 집중 분석 |
| **ASO 점수** | 항목별 100점 만점 평가 |
| **개선 우선순위** | 효과 크기 기준 실행 순서 |
| **아이콘 피드백** | 장르 트렌드 대비 개선 방향 |

### (선택) 편집장 메시지
- 임재청 편집장의 환영/격려 메시지
- 게임에 대한 개인적 감상
- ASO 전문 조언이 아닌 관계성 메시지
- 생략 가능

Day 3-4: QC & 검수
├── [현재] 관리자 백오피스(/admin)에서 결과물 확인
├── [현재] 상태 전환 · 자료 미리보기

Day 5: 전달
├── [런칭 목표] 고객 대시보드 업로드 (현재 미구현, 관리자 수동 전달)
├── [런칭 목표] 이메일 알림 (Resend)

## 5. 스크린샷 제작 파이프라인 (composite 재설계 반영)

### 근본 원칙 (2026-04-13 재설계)

- ❌ **과거 폐기**: 고정 템플릿 선택 · AI가 캔버스 전체 그리기 · 게임 그래픽 박스에 넣어 재디자인
- ✅ **현재**: 게임 플레이 캡처는 **1080×1920 원본 전체 배경**. 우리가 디자인하는 건 **오버레이(텍스트·강조)만**. 게임 이미지는 개발자가 업로드한 그대로 사용.
- ✅ **워크플로우 분기**: 업로드 자료 평가 → (자료 부족 시) **가이드 문서 발급** OR (자료 충분 시) **제작 진입**. 무조건 제작 금지.

### 전체 흐름 [현재]

```
1. 업로드 자료 평가
    ├── judge-materials.ts — 자료 충분성·품질 판정
    ├── 장르·타겟 시장 자동 분류
    └── 비주얼 아이덴티티 추출

2. 분기
    ├── 자료 부족 → upload-guide.ts (재업로드 가이드 발급, 제작 보류)
    └── 자료 충분 → 제작 경로 진입

3. Library 커버리지 조회 (Stage 8 통합 후)
    └── library-coverage.ts — 해당 게임의 axis_key 조합 패턴 조회
                              (장르×시장×수익모델×스튜디오규모)

4. 오버레이 디자인
    ├── overlay-design.ts — 슬롯별 카피·위치·스타일 결정
    ├── 원본 스크린샷을 1080×1920 전체 배경으로 사용
    └── 텍스트·강조 오버레이만 디자인 (게임 이미지 재생성 금지)

5. Composite 렌더링
    └── composite-renderer.ts — 원본 배경 + 오버레이 합성 → PNG 출력

6. 가이드 문서 동시 생성
    └── generate.ts — 고객이 Canva 등으로 직접 제작할 수 있는 슬롯별 지침
```

### 가이드 문서 동시 제공

자동 composite 결과물과 동시에 가이드 문서도 함께 발급됨.

고객이 선택:
- 완성된 composite 이미지를 그대로 사용
- 또는 가이드 따라 Canva에서 직접 제작
- 또는 둘 다 활용 (기본 composite + 커스텀 수정)

## 8. 서비스 운영 체계

### AI 시스템이 하는 일 (고객/마케팅에서는 언급 안 함)

- 스토어 데이터 수집 (Google Play scraper)
- 장르 분류 + Library 패턴 조회 (통합 후)
- 텍스트 결과물 생성 (제목/서브타이틀/소개/키워드)
- 업로드 자료 평가 · 오버레이 디자인 · composite 합성
- 가이드 문서 생성

### 임재청 편집장이 하는 일

- 주문 접수 알림 수신
- 결과물 형식적 QC (오탈자, 명백한 오류)
- (선택) 편집장 메시지 작성
- [고객에게 전달] 버튼 클릭

**주문당 소요 시간: 10~15분**

### 정무식 발행인이 하는 일

- 주 2~3회 대시보드 확인
- 중요 고객 상담 (대형 스튜디오 등)
- Founding Partner 제안
- 월간 성과 확인

## 11. 핵심 원칙 (절대 잊지 말 것)

| 원칙 | 의미 |
|------|------|
| **바로 적용 가능** | 리포트가 아닌 완성물 (텍스트 + 스크린샷 이미지) |
| **공신력 전면** | 인디게임닷컴의 미디어 지위 + 정무식/임재청 네트워크 |
| **진실된 파트너** | "싸구려 자동화" 아님, 결과물 품질로 말한다 |
| **한국어 기반** | K-인디 우선, 해외 인디는 후속 Phase |
| **시간차 전달** | 내부 처리가 빠르더라도 5영업일 후 전달 |
| **"AI" 언급 금지** | 마케팅/고객 대면에서 내부 기술 노출 안 함 |
| **게임성 판단 아닌 ASO** | 평점·리뷰 테마 분석 금지. ASO 는 게임을 "부각" 하는 기술 |
| **Library 우선 · 온디맨드 확장** | 의뢰마다 경쟁작 Vision 재호출 금지. Library 조회 후 부족 시 관리자 승인 확장 |

### 정무식/임재청의 실제 역할 (Phase 1 ASO)

ASO 서비스에서 두 분은 **ASO 실무 전문가가 아니다**.
- ASO 분석·생성은 내부 시스템이 수행
- 두 분의 역할: **공신력 + 게임 업계 네트워크 + 형식적 QC**
- 편집장 코멘트는 "환영·격려" 수준 (ASO 조언 아님)
- 주문당 업무 시간: 10~15분

Phase 2 보도자료 서비스에서는 다르다 — 임재청의 기자 경력·반다이남코 경력·미디어 네트워크가 실질 가치.

### 지식 체계
- `docs/aso/knowledge.md` = ASO 원리·하드룰·축별 변주의 Source of Truth
- `lib/aso/principles.ts` = 기계 판독용 상수 (knowledge 와 동기화). 주문 분석 프롬프트에 주입

### Reference Library 3층 구조
- **관찰층** — `reference_games` + `reference_screenshots` (L1: 아이콘·텍스트·슬롯 분석)
- **패턴층** — `library_patterns.patterns` 집계 필드 (L3 합성)
- **인사이트층** — 같은 JSONB 의 decision_rules · edge_cases · anti_patterns · cross_axis · commission_derived
- 상세: `docs/12-library-analysis-design.md`

### 수집 원칙 — IP·AAA 필터
매출 상위라도 **IP·브랜드 파워로 매출 나오는 게임은 제외** (리니지·Pokemon·Brawl Stars 등). ASO 혁신 사례 (Dream Games·Playrix·Voodoo·Century 등) 만 수집 대상.

### 의뢰 처리 흐름 (Stage 8 Library 통합 완료, 2026-04-14)

```
의뢰 접수 → 분류 (장르·시장. 수익모델·스튜디오 규모는 선택)
  → 의뢰 게임 스크랩·분석 (스토어 페이지·업로드 자료)
  → Library 조회 (pattern-query.ts):
      - 주축 primary_pattern: axis_key fallback (specific_4axis → genre_market_monetization → genre_market → genre_only)
      - 유사 게임 similar_games: 같은 장르·시장의 L2 완료 reference_games 최대 3개 (보강 참고용)
  → 경쟁작 실시간 수집 (Library 주축 있으면 3, 없으면 5)
  → 가이드 합성 (Opus):
      우선순위 = Library 주축 > 유사 게임 > 경쟁작 > 장르 벤치마크
  → 산출물 (텍스트·스크린샷 가이드·리포트)
  → [미구현 / A-2 가드로 차단] 확장 필요 판정 + 관리자 승인 + commission_driven 누적
```

**핵심 철학**
- Library 는 **항상 존재**하는 **종합 ASO 기준 축** (1:1 매칭이 아닌 합성 기준)
- `fallback_level` 은 주축 **매칭 세밀도** 메타 (무매칭이 아님). 내부용, 결과물 문구 노출 금지
- 유사 게임은 **보강 참고**, 1:1 모방 금지
- 경쟁작 실시간은 "Library 만으로 부족할 때 가이드 품질을 더 높이기 위한 추가 분석 소스"
