# ASO 지식 — 서비스 적용 기준

> 내부 지식 문서. 인디게임닷컴 ASO 자동화 서비스의 판단 근거.
> 대상: 인디 1차 · 일반 게임사 2차. 게임 한정. Google Play + Apple App Store. 한·미·일·중 시장.
> 표기 규칙: 핵심 주장에만 본문에 `(출처 · YYYY-MM)`. 수치는 출처·날짜 필수. 표본 없는 "대부분 / 100%" 금지.
> 원자료는 `raw-notes/` 참조. 최종 적용 로직은 `principles.ts` 와 함께 관리.

---

## 1. ASO 가 최적화하는 것

ASO 는 스토어에서 **① 검색·발견에 노출되는 것** 과 **② 노출 후 설치로 전환되는 것** 두 가지를 동시에 최적화. 이 둘은 상호 강화 관계다 — 랭킹이 올라갈수록 트래픽이 늘고, CVR 이 높을수록 랭킹에 긍정적 시그널이 쌓인다.

### 1.1 Apple vs Google Play 구조 차이 (핵심만)

| 항목 | Apple App Store | Google Play |
|---|---|---|
| 검색 랭킹 인자 (공식) | title · subtitle · **keywords (비공개 100자)** · primary category · downloads · ratings · reviews | title · short description · full description · category · **설치·리텐션·크래시 등 Vitals** |
| 인덱싱 범위 | 명시된 텍스트 필드만 | 모든 메타데이터 텍스트 |
| Subtitle | 있음 (30자, 랭킹 인자) | **없음** (short description 80자 가 유사 역할) |
| Promotional text | 있음 (170자, **랭킹 영향 없음 — Apple 공식 명시**) | 없음 |
| Description 한도 | 공식 페이지 숫자 미기재 (업계 통용 4,000자) | 4,000자 공식 |
| A/B 테스트 | Product Page Optimization (유료 없이 내장, 최대 3 treatment · 90일) | Store Listing Experiments (무료 내장) |
| 맞춤 페이지 | Custom Product Pages (최대 70개, 키워드 할당 가능) | Custom Store Listings (최대 50개, 국가·URL·키워드·사용자 상태 타겟) |
| 이벤트 | In-App Events (동시 10개, 승인 15개) | LiveOps cards (time-limited 최대 30일) |

- Apple 공식: 검색 결과는 텍스트 관련성(title/subtitle/keywords/primary category)과 유저 행동(downloads/ratings/reviews) 기반 (Apple Developer · 2026-04 확인)
- Google Play 공식: 기술 품질(Android Vitals) 과 engagement(리텐션·크래시 율) 가 Play 노출 기회에 영향 (Android Developers Blog · 2022, 재확인 2026-04)

### 1.2 스토어 외 발견 채널
스토어 내부 트래픽이 전부가 아니다. 시장별 편차가 큼(→ §4.2 Markets).
- **중국**: 쇼트비디오(Douyin·Bilibili) 게이머 유입 41%, 스토어 검색 38.7% (Niko Partners · 2025)
- **미국·일본·한국**: 유튜브·트위터·디스코드·커뮤니티 경유 인디 유입 상당수. ASO 단독으로 인디 성공은 드묾.
- 신생 AI 검색 경로: Google Play "Guided Search" 2025-09 정식, "Ask Play" 명칭은 공식 확인 안 됨 — 모니터링 대상.

## 2. 하드 룰 — 반드시 지켜야 할 것 (장르·시장 무관)

### 2.1 필드 한도

| 필드 | Apple App Store | Google Play |
|---|---|---|
| Title (app name) | 30자 | 30자 |
| Subtitle | 30자 | — (없음) |
| Short description | — | 80자 |
| Promotional text | 170자 (버전 제출 없이 수시 변경 가능) | — |
| Keywords field | 100자 (콤마 구분, 비공개 메타) | — |
| Description | 공식 숫자 미기재 (통용 4,000자) | 4,000자 |
| In-App Event name | 30자 / short 50자 / long 120자 | 이벤트 description (유형별) |

출처: Apple Developer "Creating Your Product Page" · App Review Guidelines 2.3.7 · Google Play Console Help 9866151 / 13393723 / 9898842 — 2026-04 확인

### 2.2 이미지·영상 스펙

**아이콘**
- Apple: 1024×1024 PNG, no-alpha (업계 통용 / HIG 직접 확인 실패 — JS 렌더 페이지. 재확인 필요)
- Google Play: **512×512 PNG 32-bit (알파 포함), 최대 1024 KB** (Google Play Console Help · 2026-04)

**스크린샷**
- Apple: 디바이스당 1~10장. iPhone 6.9" 1290×2796 또는 2796×1290, iPad 13" 2064×2752. JPEG/PNG. (Apple Developer · 2026-04)
- Google Play: 각 디바이스 타입당 최대 8장(TV 포함 시 TV 1장 필수). 권장 8:5, 최소 1920×1200, 최대 8 MB/장. (Google Play Console Help · 2026-04)

**영상**
- Apple App Preview: **로케일·디바이스당 최대 3개**, 15~30초, 최대 500 MB, 음소거 자동재생. H.264 10~12 Mbps. (Apple Developer · 2026-04)
- Google Play Promo Video: YouTube 공개 링크 1개, Shorts·Live 불가, 수익화 OFF. (Google Play Console Help · 2026-04)

**Feature Graphic (Google Play 전용)**
- 1024×500, JPEG 또는 24-bit PNG (알파 없음). 프로모션·피처링 진열에 노출.

### 2.3 정책 제약

**양 스토어 공통적으로 금지**
- 키워드 스터핑, 경쟁 앱명 사용, 무관 키워드 반복
- 성과·랭킹·가격·프로모션 문구를 title/icon/developer name 에 삽입 (Google Play: 명시 · 2026-04)
- misleading · deceptive 메타데이터

**Apple 전용 포인트**
- 스크린샷은 "app in use" 증명 (App Review Guidelines 2.3.3)
- App Preview 는 **앱 화면 캡처만**, 손가락·디바이스 촬영 금지 (2.3.4)
- 타 플랫폼 이름·아이콘 노출 금지 (2.3.10)
- Promotional text 에 키워드 목적 스터핑 금지 — 랭킹 영향 없음

**Google Play 전용 포인트**
- Title·short·full description 에 emoji/emoticon/반복 특수문자 금지
- Short description 을 full description 에 복붙 금지
- promo video 에 손가락·디바이스 촬영 원칙 금지(오프-디바이스 게임 제외)

### 2.4 기본 위생 (범용 수칙)

1. **첫 3장 스크린샷**이 검색 결과 노출에 가장 큰 영향. 코어 가치·주요 플로우를 여기서 전달. (Apptweak · 2025)
2. **캡션 텍스트**의 랭킹 인덱싱 여부는 Apple 공식 명시 없음(통념과 다름). 다만 CVR 에는 유의미 — 음소거 자동재생 환경·시각 장애 접근성 관점에서도 필수.
3. **평점 4.0 이상**이 피처링 허들 (Apple 피처드 앱 90%, Google Play 85% · Apptweak 2025). 인디 최소 품질선.
4. **리뷰 응답**: Google Play 응답 앱 평균 평점 +0.7★ 상승 (AppReply · 2024, marketing_claim 유의). 두 스토어 모두 응답 기능 무료 제공.
5. **A/B 테스트**: 두 스토어 모두 무료 내장 도구 제공. 분기 이상 주기로 최소 스크린샷 테스트. 월 단위 미만은 인디에 노이즈 지배(heuristic).

## 3. 공통 원리 — ASO 가 먹히는 이유

### 3.1 유저 검색 행동

- **이름 검색 vs 장르 검색 2분화**: 대형 IP 는 이름 검색(숏테일), 인디는 장르·상황·기능 기반 롱테일 ("match 3 no ads", "방치형 RPG", "relaxing offline"). 숏테일은 AAA 점유 → 인디는 반드시 롱테일 공략 (Appfigures · 2025).
- **3~4 단어 롱테일 최적화**: 경쟁도 낮고 의도 명확 → 전환 우위. 일부 벤치마크상 60일 내 오가닉 설치 +32% 보고 (MobileAction · 2026, marketing_claim 유의).
- **언어 구조별 검색 단위**: 일본 유저는 **히라가나·카타카나·한자·영문 혼합** 으로 다표기 인덱싱 필요. 한국 유저는 한글 + 장르 영문 병기. 중국 유저는 한자 직접 타이핑 (병음 검색 드묾). (GameRefinery · AppTweak 일본·중국 가이드 2024~2025)

### 3.2 스토어 알고리즘 시그널 (공식 언급 기준)

**Apple 공식**
- 텍스트 관련성: title / subtitle / keywords / primary category 인덱싱
- 유저 행동: downloads / ratings / reviews
- 명시적으로 배제: promotional text

**Google Play 공식**
- 텍스트: title · short description · full description · category
- 품질·engagement: Android Vitals(크래시·ANR·배터리 등), 리텐션, 언인스톨율, 평점

**공통 비공식 해석 (Tier 3, 참고)**
- CPP/CSL 가 오가닉 검색에 영향을 주는지는 스토어 공식 수치 없음. Apple 은 키워드 할당으로 검색 결과 교체 가능 명시(CPP 한정).
- In-App Events / LiveOps 는 노출 카드·검색 결과 카드로 이벤트 페이지 추가 트래픽을 만들지만, 랭킹 직접 가중치는 비공개.
- **스크린샷 캡션 인덱싱 설**: 통념이 있으나 Apple 공식 문서에서 확인 안 됨. CVR 효과는 별개로 유효.

### 3.3 전환 결정 경로 (노출 → 설치)

```
스토어 검색/추천 → 결과 썸네일(아이콘 + 제목 + 첫 1~3장) 
  → 상세 페이지(스크린샷 전체 + 영상 + 설명 첫 250자 + 리뷰/평점) 
  → 설치 결정
```

- 썸네일 단계에서 약 절반 이상이 걸러짐(벤더 추정, marketing_claim) — **아이콘 + 첫 3장이 CVR 의 입구**
- 상세 페이지에서 영상 재생은 CVR +20~40% 보고 다수 (Hilomedia · 2025, Storemaven 등 — 벤더별 marketing_claim 유의)
- 첫 250자(App Store 는 더보기 이전 노출, Google Play 는 short + full 앞부분)가 설명 전환 좌우
- 평점이 낮으면 결정 경로 끊김: 3.6 → 4.2 개선 시 CVR 약 +60% (Apptweak · 2025)

### 3.4 게임 카테고리의 특수성

- 게임 평균 CVR: 약 3~5% 목표 수준(Apptweak · 2025). 전체 앱 평균(Apple 25% / Google Play 27.3%, Adapty · 2026 정리)보다 훨씬 낮음 — 경쟁 치열·선택지 많음.
- 서브장르 편차 극심(Racing 20.6% vs Board 1.2% App Store — 측정 기준 차이 가능).
- Google Play 게임의 **57% 가 스크린샷 A/B 테스트 2회/년 이상** 실행 (비게임 앱 34%) — 테스트 문화가 앱보다 활발 (Apptweak · 2025).

## 4. 축별 변주 — 가이드의 실제 분기

### 4.1 장르 매트릭스 (프레임)

> 구체적 패턴(예: "퍼즐 아이콘은 보라·골드")은 이 문서에 고정 기록하지 **않는다**. 장르 패턴은 시대·시장에 따라 바뀌므로 Reference Library 의 **귀납 관찰 결과**에서 조회한다. 본 문서는 "어떤 축에서 장르 차이가 나는가"의 프레임만 제공.

**장르별로 관찰해야 할 축**
- 아이콘 컴포지션 (캐릭터 얼굴 vs 오브젝트 vs 로고 vs 게임플레이 요소)
- 아이콘 색·분위기 (다크·프리미엄 · 밝은·캐주얼 · 네온·아케이드 · 따뜻·코지)
- 스크린샷 시퀀스 공식 (첫 3장의 구조 — hook / 코어 메카닉 / 차별화 / 사회적 증거 / CTA)
- 스크린샷 세로·가로 비중
- 서브타이틀·short_description 패턴 유형 (장르 키워드형 / 소셜·멀티형 / 감성·서사형 / 최상급 주장형 / 메카닉 설명형 / IP·캐릭터형)
- 소개문구 첫 250자 훅 기법 (질문 · 혜택 나열 · 스토리 · 기능 · 사회적 증거)
- 영상 스타일 (시네마틱 · 게임플레이 · UGC 풍 · 튜토리얼)

**장르 분류 기준**
- Google Play / Apple 의 공식 카테고리를 기본으로
- **교차 장르(예: Empires & Puzzles = 퍼즐+RPG)는 1차 장르 + 2차 장르 태그** 로 다룬다. 1차 = 대표 코어 루프 기준.
- 서브장르(Battle Royale, MOBA, Auto-Chess, Roguelike 등)는 태그로만 추가.

### 4.2 시장 축 — 한·미·일·중

> 국가별 상세는 각 소섹션. 수치는 분기·연도별 변동성이 크므로 인용 시 **시점 명시**.

#### 한국 (KR)
- **플랫폼**: Android 매출 75% / iOS 25% (Sensor Tower · 2025 H2). 1차 Google Play, 고ARPU·얼리어답터는 iOS.
- **키워드**: 한글 기본 + 장르 영문 병기 흔함 (예: "방치형 RPG", "MMORPG"). 숏테일은 대형 IP 점유, 인디는 롱테일 필수.
- **비주얼**: MMORPG 는 캐릭터 풀샷 + 금·진홍 톤. 캐주얼·하이퍼캐주얼은 글로벌 템플릿 수용. 스크린샷 첫 3장에 한글 카피 필수.
- **서사·감성**: 미드코어는 경쟁·스펙트럼("서버 최강"), 캐주얼·방치형은 실용("광고 없이", "오프라인").
- **수익모델**: F2P+가챠+배틀패스 지배. RPG 매출 52% (2024 1~10월, Sensor Tower). Premium 선불은 약함.
- **장르 편향**: Idle RPG > MMORPG > 소셜카지노 > 전략 > 퍼즐. 인디 기회는 방치형 하이브리드·힐링 퍼즐·비주얼노벨.
- **규제**: GRAC 등급 필수. 확률형 아이템 공시 의무(2024-03 시행).
- **현지화**: **번역기 티 즉시 거부** — 네이티브 카피 요구(일본과 동급).
- **리뷰 문화**: Google Play 리뷰 활발. 3점대는 의심, 4.3+ 기본 기대. 집단 별점 하락 빠름.
- **스토어 외**: 유튜브·유튜브 쇼츠·나무위키·루리웹·인벤 커뮤니티 경유 비중 큼.

#### 미국 (US)
- **플랫폼**: 양대 스토어 모두 1차 타깃. 다운로드 Google Play 우위, 매출 iOS 우위(업계 통용, 공식 숫자 없음).
- **키워드**: 영어 생태계 최대 경쟁. 장르·기분 검색 ("city builder", "match 3", "relaxing offline"). 숏테일 AAA 점유 → 인디는 롱테일+negative keyword ("match 3 no ads").
- **비주얼**: 하이퍼캐주얼은 Voodoo/Zynga 템플릿 수준의 간결·대비. 미드코어·RPG 는 폴리싱 3D. 스크린샷 텍스트 **여백 많고 짧음**(중·일 대비) — 기능 1개 = 스크린샷 1장.
- **서사·감성**: "Best", "#1", "Million Players" 최상급 허용(App Review 2.3 범위 내). 커뮤니티 훅("Play with friends") 효과.
- **수익모델**: F2P 주류, 광고 수용도 세계 최고. 구독 점진 성장.
- **장르 편향**: Puzzle (Block Blast 356M 설치 2025), Roblox, .io 계열, Battle Royale. Match-3 · Merge · Sim 성숙. 인디 기회: 내러티브 퍼즐·코지 게임.
- **규제**: COPPA, ATT, CCPA. 주별 loot box·AI 표기 법안 증가.
- **리뷰 문화**: 작성 빈도 매우 높음. 리뷰 응답 문화 정착 — 무응답 감점 요인.

- **장르 편향**: RPG(가챠형·JRPG) 압도, 퍼즐(P&D 장기 집권), 리듬, 아이돌·캐릭터 수집, 방치형. BR·FPS 는 상대적으로 약함.

#### 시장 설계 원칙 (서비스 전용)
1. **중국은 "언어 확장"이 아니라 "별도 서비스 트랙"**. 판호·서드파티·쇼트비디오 모두 달라서 4개 시장 균일 템플릿 불가.
2. **한·일은 번역기 티 = 즉시 실패**. Tier 1 번역 파이프라인은 기계 번역 후 네이티브 리라이트 필수.
3. **일본 가챠 규제는 카피 레벨에서도 체크**. 생성 프롬프트에 JP 타깃 시 금칙어 가드레일.
4. 시장 차트·매출 수치는 분기·연도별 변동성 크므로 **시점 명시**하여 인용.

### 4.3 수익모델 축 (F2P 광고 · F2P IAP · 구독 · Premium · Hybrid)

#### F2P + 광고 (하이퍼캐주얼 / 라이트 퍼즐)
- 제목·서브에 "Free" 쓰지 않음(이미 표시). 장르·액션 동사("Tap", "Match", "Run") 중심.
- 첫 3장에 **결과 화면(점수·연쇄 폭발)** 배치해 도파민 유발.
- 아이콘: 밝은 원색·과장 표정·결과 액션 노출.
- 리뷰의 구조적 불만: 광고 빈도. 리뷰 응답에 "Remove ads IAP" 안내 표준.
- ASO 전략: **설치 볼륨 최대화 우선**. 텍스트보다 시각 훅.

#### F2P + IAP (RPG · 전략 · 미드코어)
- 제목·서브에 과금 언급 없이 세계관·장르·사회적 훅("Clan Wars", "Guild", "Co-op") 배치.
- 스크린샷: 캐릭터 수집·장비 강화·레벨업·PvP. 캡션 "Collect / Upgrade / Build your team".
- 아이콘: 캐릭터 클로즈업(IP 인지 또는 톤 전달). 시즌 교체는 AAA 전용.
- 리뷰의 구조적 불만: "Pay to win", "확률 조작".
- ASO 전략: 설치 양 + 질 동시. 엔드게임 플레이어 유입이 LTV 좌우. D7/D30 리텐션 지지 크리에이티브 우선.

#### F2P + 구독 (Apple Arcade · Google Play Pass)
- Apple Arcade 는 제목 끝 `+` 관례. 서브 "No Ads. No IAP." 흔함.
- 스크린샷은 시네마틱·아트 퀄리티·다양성. 수익 설득 카피 거의 없음.
- ASO 비중 낮음. 트래픽 70%+ 가 큐레이션 피처(추정, 공식 비공개).
- **본 서비스 우선순위 낮음** (큐레이션 게임 특성).

#### Premium 유료 (Monument Valley · Retro Bowl · Balatro 등)
- 서브에 "Full game", "Pay once", "No ads. No IAP." 표준. Apple App Store 의 "Pay Once & Play" 섹션 노출이 주요 이벤트.
- 스크린샷: 아트 디렉션·내러티브·퍼즐 구조 중심. "Award-winning / Critically acclaimed / Hand-drawn" 품질 수식어.
- 아이콘: 아트북 표지 느낌. 브랜드 일관성 강함(시즌 교체 거의 없음).
- CVR 낮음(가격 장벽) → 첫 3장 + 영상 + 리뷰 문구로 **가치 설득** 최적화.
- 외부 커뮤니티·스트리머 오버랩이 스토어보다 강한 드라이버인 경우 다수(Balatro 사례).

#### Hybrid (무료 + Remove Ads IAP · 배틀패스 등)
- F2P+IAP 설계 기본, 상점 카피에 "ad-free option available" 명시 → 리뷰 리스크 감소.
- 2025 이후 "광고+IAP+구독+배틀패스" 병행이 상위 매출 공통(ASOMobile · 2025).

### 4.4 스튜디오 규모 축

| 규모 | 현실적 상한 | AAA 답습 시 손실 |
|---|---|---|
| **Solo (1인)** | 스크린샷 10장·CPP 1~2개·영상 1편·Tier 1 로컬(EN/KR/JP/CN). 리뷰 응답 템플릿+주 1~2회 배치. | 시즌 아이콘 교체·유료 CPP 광고 매칭·월간 A/B 로테이션 — 표본 부족 노이즈 + 예산 소진 |
| **Indie (2~20인)** | CPP 3~5개·외주 트레일러 가능·Google Play Reply API 자동화 임계점 | 위와 동일 + 매주 A/B 로테이션 비현실적 |
| **Mid (중견)** | CPP 3~5 · Apple Search Ads 적정 예산 · Tier 2 로컬(DE/FR/TW) · 24시간 CS · A/B 스토어+광고 동시 | 여전히 **매주 아이콘 교체는 리스크**(브랜드 일관성) |
| **AAA / IP** | 제목 = 검색 볼륨. 시즌마다 전면 교체. 30+ 언어. 전담 CS. | 본 서비스 대상 아님 |

**인디 적용 가능성 태그 규칙** (서비스 출력에 항상 명시)
- `indie_not_applicable`: AAA 전용 — 시즌 아이콘 교체 · 유료 CPP 광고 매칭 · 30+ 언어 동시 · 월 단위 미만 A/B · 대형 IP 콜라보 피처링 · 매주 아이콘 교체
- `indie_recommended`: 특히 인디에 중요 — 장르 롱테일 키워드 · 첫 3장 hook 품질 · 리뷰 응답 템플릿화 · 장르 오버랩 스트리머 타겟(heuristic)
- `all_sizes`: 전 규모 적용 — 하드 룰 / 기본 위생 / CVR 최적화 원칙

### 4.5 축 충돌 시 우선순위

| 충돌 | 우선 | 근거 |
|---|---|---|
| 장르 공식 vs 의뢰 게임 고유 경쟁력 | **고유 경쟁력** | 장르 공식은 일반 가이드일 뿐, 기계 적용 금지 |
| 시장 선호 vs 장르 공식 | **시장 선호** | 같은 퍼즐도 일본·중국·미국에서 통하는 아이콘·카피가 다름 |
| 수익모델 vs 장르 공식 | **수익모델** | "광고 없음" 카피를 F2P+광고 게임이 쓸 수 없음 |
| 스튜디오 규모 vs 나머지 | **규모 제약이 상한선** | 실행 불가 전략을 권장하면 가이드 품질 훼손 |

## 5. 의뢰 게임 적용 의사결정

### 5.1 분류 체크리스트 (서비스가 먼저 판정할 것)

1. **장르 (1차 + 2차 태그)** — 주문 정보·스토어 URL·업로드 자료로 판정
2. **타겟 시장** — 주문 선택값. 중국 포함 시 **별도 트랙** 로 플래그
3. **수익모델** — 스토어 IAP 목록·가격·"Remove Ads" 존재 여부·광고 SDK 흔적으로 휴리스틱 판정. 개발자 제공값 우선
4. **스튜디오 규모** — 팀 크기·과거 타이틀·예산 입력으로 Solo/Indie/Mid/AAA 판정

### 5.2 변주 조합 규칙

```
(장르 프레임) × (시장별 선호) × (수익모델 방향) × (규모 제약)
  → 항목별 권장안 (제목·서브·아이콘·스크린샷·소개문·영상)
  → 각 권장안에 (근거 · 인디 적용도 · 예외 조건) 태깅
```

- 각 항목은 **"왜 이 권장인가"의 증거** 를 반드시 첨부(랭킹 인자·시장 사례·Library 패턴).
- AAA 전용 전략은 `indie_not_applicable` 로 자동 필터.
- 시장별 규제 금칙어는 **카피 생성 프롬프트 가드레일** 로 주입(JP 경품표시법·KR 확률 공시·CN 신광고법).

### 5.3 공식을 의도적으로 깨는 케이스

- 장르 공식과 게임 고유 재미가 충돌할 때(퍼즐 RPG 하이브리드, 힐링 JRPG 등) — **고유 경쟁력을 1차 포지셔닝**으로 삼고 장르 공식은 참고자료로만
- 시장이 포화된 키워드를 의도적으로 버리고 **2차 타겟 롱테일** 잡기 (예: 매치 3 대신 "coloring match")
- Premium 이지만 "가격"을 전면 배치하지 않고 **가치 + 커뮤니티** 로 설득 (Balatro 류)

**예외 판단 규칙**: 의뢰 게임의 고유 경쟁력에 대한 근거(리뷰 정성·장르 오버랩 커뮤니티 반응·실제 플레이 영상 등)가 **2개 이상** 있을 때만 공식을 깬다. 막연한 "특별해서"는 근거 아님.

### 5.4 최종 산출물 구조 (각 의뢰당)

항목별로 아래 4-part 구조를 반환.

```
- 권장안 (구체적 문안 / 이미지 레퍼런스 / 시퀀스)
- 근거 (장르 프레임 / 시장 사례 / Library 관찰 / 하드 룰)
- 인디 적용도 (indie_recommended | all_sizes | indie_not_applicable)
- 예외·대안 (이 권장을 버리는 게 낫다면 어떤 상황인가)
```

적용 대상:
- **제목** (양 스토어 공통 / 필요 시 시장별 분기)
- **서브타이틀**(Apple) + **short description**(Google Play)
- **소개문구** (첫 250자 훅 + 전체 구조)
- **아이콘** (레퍼런스 + 변주 제안)
- **스크린샷 시퀀스** (첫 3장 hook · 중반 차별화 · 후반 사회적 증거 · 캡션 원고)
- **영상** (길이·첫 몇 초 hook·캡션 전략)
- **In-App Events / LiveOps** (해당 시)
- **Keywords field**(Apple) — 100자 내 전략 키워드 세트
- **CPP/CSL 제안** (규모·예산 조건 맞을 때만)

6. **Description 문자 상한 Apple 공식 수치** — 재조사
