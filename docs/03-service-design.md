# 서비스 상세 설계

## 서비스 A: 보도자료 제작 & 배포

### 현재 워크플로우 (수동)

```
보도자료 수신 (이메일) → 편집장 기사 가치 판단 → 편집장 스타일로 재구성
→ 한글 기사 포스팅 → 영어 번역 → 영문 포스팅
```

### 자동화 목표 워크플로우

```
[인디 개발자가 게임 정보 입력]
    ↓
[AI 1단계] 보도자료 초안 자동 생성
    - 편집장 스타일 시스템 프롬프트 적용
    - 게임 장르/특성에 맞는 톤 자동 선택
    - 한글 보도자료 초안 생성
    ↓
[AI 2단계] 다국어 편집 번역
    - 한→영 편집 번역 (직역 아님)
    - 게임 업계 용어 DB 반영
    - 영문 독자 맥락에 맞게 재구성
    - (옵션) 일본어, 중국어 번역
    ↓
[편집장 검수]
    - 모바일/웹에서 검수 UI
    - 승인/수정 → 최종본 확정
    ↓
[자동 배포]
    - indiegame.com 자동 게시 (한글/영문)
    - 200+ 매체 이메일 배포
    - 성과 트래킹 (열람률, 클릭률)
```

### 입력 폼 설계

```
필수 입력:
- 게임 제목 (한글/영문)
- 개발사/스튜디오명
- 장르
- 플랫폼 (PC/모바일/콘솔)
- 출시일 또는 예정일
- 게임 소개 (300자 이상)
- 핵심 특징 3~5가지
- 스크린샷/트레일러 URL

선택 입력:
- 스토어 URL (Steam, App Store, Google Play)
- 이전 보도자료 샘플
- 희망 톤 (공식적/캐주얼/기술적)
- 타겟 지역/언어
```

### 편집장 스타일 학습 방법

```
1. 기존 인디게임닷컴 기사 100건 수집 (대표적 잘 쓴 기사)
2. 패턴 추출:
   - 리드문 구조 (주어-동사-목적어 패턴)
   - 인용 방식 (개발자 인터뷰 스타일)
   - 맺음말 패턴 (기대감/전망)
   - 자주 쓰는 표현 / 피하는 표현
3. Few-shot 예시로 시스템 프롬프트 구성:
   [보도자료 원문] → [편집장이 쓴 기사] 쌍 10~20개
4. 피드백 루프:
   편집장 수정 → 수정사항을 프롬프트에 반영 → 반복
```

### AI 프롬프트 구조

```python
SYSTEM_PROMPT = """
당신은 인디게임닷컴의 수석 편집자입니다.

## 톤 & 스타일
- [기존 기사에서 추출한 톤 특성]
- [리드문 구조: OO하는 OO이(가) OO했다]
- [본문: 특징 나열 시 불릿 포인트 활용]
- [맺음말: 기대감 + 출시/이벤트 일정 언급]

## 규칙
- K-인디게임 생태계 활성화 관점 유지
- 개발자의 노력과 창의성을 존중하는 어조
- 과대 광고 지양, 사실 기반 서술
- 게임의 독특한 점을 먼저 부각

## 구조
1. 제목 (30자 이내, 핵심 뉴스 포인트)
2. 리드 (1문장, 누가-무엇을-언제)
3. 본문 (게임 소개 → 특징 → 개발 배경)
4. 출시/이벤트 정보
5. 스튜디오 소개 (보일러플레이트)
"""
```

## 서비스 B: 게임 콘텐츠 번역

### 번역 유형별 파이프라인

| 유형 | 입력 | 처리 | 출력 |
|------|------|------|------|
| 스토어 소개 번역 | 한글 스토어 설명 | AI 편집 번역 + ASO 키워드 반영 | 영문 스토어 설명 (ASO 최적화) |
| 보도자료 번역 | 한글 보도자료 | AI 편집 번역 (직역 아님) | 영문 보도자료 |
| 게임 내 텍스트 | 스프레드시트/JSON | AI 번역 + 게임 용어 DB | 다국어 텍스트 파일 |
| UI/UX 텍스트 | 키-값 쌍 | AI 번역 + 길이 제한 준수 | 로컬라이즈된 리소스 |

### 번역 품질 보장 체계

```
[AI 1차 번역]
    ↓ 게임 용어 사전 적용
[AI 2차 검수] — 자연스러움, 문법, 맥락 확인
    ↓
[LQA 체크리스트]
    - 맞춤법/문법 오류 없음
    - 게임 용어 일관성
    - 문화적 적절성 (지역별)
    - 텍스트 길이 제한 준수
    - 플레이스홀더/변수 보존
    ↓
[편집장 최종 검수] (프리미엄 플랜)
```

### 스토어 소개 번역 시 ASO 연계

```
한글 소개 → 영문 편집 번역
                ↓
         ASO 키워드 리서치
                ↓
         키워드를 자연스럽게 소개문에 삽입
                ↓
         첫 252자에 핵심 키워드 집중 배치
```

**핵심 원칙**: 번역기 사용 금지. 전문 편집 번역으로 영문 독자가 자연스럽게 읽을 수 있는 품질.

```
나쁜 예 (번역기):
"이 놀랍도록 화려하고 상큼팍팍 가슴을 통쾌하게 후려치는 퍼즐게임"

좋은 예 (편집 번역):
"Move sparkling jewel blocks to match three of a kind in this colorful puzzle adventure."
```

## 서비스 C: ASO 최적화

### ASO 자동 분석 서비스 흐름

```
[개발자 입력]
    - 게임 제목 (현재)
    - 장르
    - 스토어 URL 또는 스크린샷
    - 아이콘 이미지
    - 서브타이틀
    - 소개문구
    ↓
[자동 분석 엔진]
    - 장르 자동 분류 (9개 장르)
    - 해당 장르 Top 10 벤치마크와 비교
    - 각 항목별 점수 산출 (100점 만점)
    ↓
[분석 리포트 자동 생성]
    ├── 제목 분석 (25점)
    │   - 키워드 포함 여부
    │   - 길이 적정성
    │   - 콜론/하이픈 구조
    │   - 브랜드 vs 제네릭 밸런스
    │
    ├── 서브타이틀 분석 (15점)
    │   - 6가지 패턴 중 분류
    │   - 장르별 효과적 패턴 매칭
    │   - 키워드 밀도
    │
    ├── 아이콘 분석 (20점)
    │   - 색상 팔레트 → 장르 벤치마크 비교
    │   - 구도 (캐릭터/사물/로고)
    │   - 작은 사이즈 가독성
    │   - 경쟁작 대비 차별화
    │
    ├── 스크린샷 분석 (25점)
    │   - 장수 적정성 (장르별 권장 대비)
    │   - 방향 (세로/가로)
    │   - 텍스트 캡션 유무 (Apple 랭킹 영향)
    │   - 첫 3장 품질
    │   - 프리뷰 영상 유무
    │
    └── 소개문구 분석 (15점)
        - 첫 252자 핵심 정보 밀도
        - 장르별 필수 키워드 포함
        - 전환율 부스터 키워드 ("no ads", "offline")
        - 영문 품질 (번역기 감지)
    ↓
[개선안 자동 생성]
    - 제목 대안 3개 제안
    - 서브타이틀 대안 3개 제안
    - 아이콘 디자인 방향 제안
    - 스크린샷 구성 가이드
    - 소개문구 개선안 (첫 252자)
    - 키워드 추천 목록
    ↓
[편집장 검수] (프리미엄)
    - AI 분석 결과 리뷰
    - 추가 인사이트 코멘트
    - 최종 리포트 전달
```

### 장르별 벤치마크 DB

> 상세 데이터: `ASO-bible-by-genre-2026.md` 참조

9개 장르, 80+ 게임의 다음 데이터 보유:
- 제목 / 서브타이틀 키워드
- 아이콘 색상 팔레트 및 구도
- 스크린샷 장수, 방향, 오버레이 스타일
- 소개문구 첫 252자 패턴
- 평점 / 다운로드 벤치마크

**업데이트 주기**: 분기별 Top 10 게임 변동 반영

### ASO 점수 등급 체계

| 등급 | 점수 | 의미 |
|------|------|------|
| S | 90~100 | Top 10 수준, 유지 권장 |
| A | 75~89 | 경쟁력 있음, 미세 조정 |
| B | 60~74 | 개선 필요, 구체적 액션 제시 |
| C | 40~59 | 주요 개선 필요, 재작업 권장 |
| D | 0~39 | 전면 재작업 필요 |

```
Day 1-2: 개발자 정보 입력 완료

         - 한글/영문 보도자료
         - 영문 스토어 소개
         - 키워드 추천 목록
Day 8~:  보도자료 200+ 매체 배포
Day 14:  배포 성과 리포트 전달
```
